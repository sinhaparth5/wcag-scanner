## Twitter / X Post Variants

### Variant 1
Built `wcag-scanner` to make accessibility testing easier during development, not just after release.

It scans for WCAG 2.1 issues, gives fix suggestions, works with a React dev overlay, supports Node scripts, and can even scan HTML responses in Express.

If you want faster accessibility checks in your workflow:
`npm install wcag-scanner`

https://www.npmjs.com/package/wcag-scanner

### Variant 2
Most teams still treat accessibility like a last-minute audit. I wanted something developers could run continuously while building.

`wcag-scanner` is my npm package for automated WCAG 2.1 scanning with:
- React dev overlay
- AI-powered fix suggestions
- HTML/file scanning from Node
- Express middleware support
- JSON, HTML, and console reports

Package:
https://www.npmjs.com/package/wcag-scanner

### Variant 3
I shipped `wcag-scanner`, an npm package for catching accessibility problems earlier in the dev cycle.

You can use it to scan HTML, run checks inside a React app with a live overlay, get AI-assisted fix suggestions for violations, and plug it into Express responses for automatic checks.

If you're building for the web and care about shipping accessible UI faster, take a look:
https://www.npmjs.com/package/wcag-scanner
